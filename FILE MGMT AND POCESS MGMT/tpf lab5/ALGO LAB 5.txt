ALGORITHM FOR PROCESS MANAGEMENT LAB 1

START
1.VALIDATE THE INPUT
          - COMPARE INPUT WITH D, IF YES THEN FIND FILE ADDRESS
                                  ELSE CHECK FOR ADD ENTRY(ENTRC PGM2)
2.FIND THE FILE ADDRESS(THROUGH FACE TABLE)
          - IF FOUND, THEN FETCH THE FILE DETAILS(USING FINWC)
                    ELSE SHOW THE RESPECTIVE ERROR
3.FETCH THE FILE DETAILS(IN DATA LEVEL 1) NOT HOLD
4.DISPLAY THE FILE DETAILS IN ITERATION UPTO THE NUMBER OF RECORDS IN FILE.
5.CHECK IF FWD CHAIN HAS ADDRESS IF YES THEN GO TO STEP 3 AND ITERATE
6.RELEASE THE BLOCK
END


PGM1

START
1.CHECK WHETHER CARD NUM IS 15 DIGITS NUM, IF NOT THEN SHOW INVALID I/P
2.CHECK EXP DATE IS PROPER AND NOT PAST, IF NOT PROPER SHOW INVALID EXPIRY
3.CHECK CARD LMIT SHOULD NOT EXCEED 99999, OTHERWISE SHOW INVALID CARD NUM
4.CHECK CARD STATUS IS EITHER A/L/S/D/E, IF NOT SHOW INVALID
5.STEP 2.
6.FETCH THE FILE DETALS WITH HOLD
7.CHECK IF FILE IS FULL(BY CHECKING THE NAB VALUE), IF IT IS, THEN CHECK FWD CHAIN IF ADDRESS IS THERE THEN ITERETE FRM STEP 12
           OTHERFICE REQUEST POOL FILE ADDRESS (GETFC) THEN ITERATE FROM STEP 12.
8.OTHERWISE FIND FOR THE EMPTY PLACE N FILE, AND UPDATE THE NAB VALUE ACCORDINGLY
9. DISP 'RECORD ADDED SUCCESSFULLY'
10.FILE AND UNHOLD THE FILE, AND DO THE STEP 5.
END
